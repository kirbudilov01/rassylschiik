# 01. Требования и входные данные (RU)

## Цель MVP
Собрать систему, которая:
1. Слушает сообщения в целевых Telegram-чатах через пользовательские аккаунты.
2. Находит релевантные запросы по теме "YouTube продюсирование".
3. Оценивает лид (AI scoring) и формирует предложение.
4. Передает сообщение на отправку с учетом лимитов и ручного контроля.
5. Ведет воронку по проекту.

## Ограничения и рамки
- Используем Telegram user accounts.
- Не делаем обход ограничений и антибан-манипуляции.
- Соблюдаем лимиты и прозрачную коммуникацию.
- В MVP отправка в личку может быть только после подтверждения оператором.

## Что нужно от тебя (owner input)

### 1) Бизнес-данные
- Название услуги: YouTube продюсирование.
- Ключевой оффер (1-2 фразы).
- Ссылка назначения: канал/лендинг/календарь.
- Примеры кейсов (3-5 шт).
- Тон общения (деловой, дружелюбный, экспертный).

### 2) Поиск лидов
- Список чатов/групп для мониторинга.
- Язык/гео для фильтрации.
- Ключевые фразы (что ищем).
- Стоп-фразы (что игнорируем).

### 3) Telegram аккаунты
- Кол-во аккаунтов на проект (пока 1).
- Для каждого аккаунта: alias, номер, session string или локальная сессия Telethon.
- Рабочие временные окна (например 10:00-20:00 МСК).

### 4) Лимиты (стартовые)
- Новые диалоги/день на аккаунт.
- Отправки/час на аккаунт.
- Отправки/день на аккаунт.
- Минимальный интервал между отправками (сек).

### 5) AI политика
- Что считать "качественным лидом".
- Порог score от 0 до 1.
- Шаблоны первого касания.
- Правила отказа (когда не писать).

## MVP Scope
1. Проекты и конфиг по каждому проекту.
2. Хранилище сообщений-кандидатов.
3. Лид-скоринг + причины.
4. Очередь outreach.
5. Human review перед отправкой.
6. Статусы: queued/sent/replied/rejected.

## Критерии готовности MVP
- Есть минимум 1 проект и 1 аккаунт.
- Можно добавлять кандидатов вручную или через API ingest.
- Кандидаты автоматически скорятся.
- В очереди видны рекомендации и предложенный текст.
- Можно принять/отклонить отправку.
- Логи действий и базовые метрики доступны в UI.
